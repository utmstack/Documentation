# Bare Metal Installation of UTMStack

## Introduction

This guide provides detailed instructions for installing UTMStack directly on physical servers. UTMStack is an open-source security platform designed for Ubuntu 22.04 LTS, featuring encrypted communications, containerized microservices, and strong authentication mechanisms. This document will guide you through the installation process using the installer file method.

## System Requirements

To ensure optimal performance and installation of UTMStack, your hardware must meet the following minimum specifications:

- **Operating System**: Ubuntu 22.04 LTS
- **CPU**: 64-bit x86-64 v2 architecture
- **RAM**: At least 16 GB
- **Disk Space**: Minimum 100 GB for federated servers
- **Cores**: At least 4 CPU cores

### Resource Recommendations for Log Storage

- **50 Data Sources (100 GB/month)**: 8 Cores, 16 GB RAM, 150 GB Disk Space
- **120 Data Sources (250 GB/month)**: 16 Cores, 32 GB RAM, 250 GB Disk Space
- **240 Data Sources (500 GB/month)**: 32 Cores, 64 GB RAM, 450 GB Disk Space

&gt; **Note**: For setups exceeding 500 data sources, additional nodes are required for horizontal scaling.

## Installation Steps

### Preparing for Installation

1. **Update Package List**:
   ```bash
sudo apt update
```bash

2. **Install WGET**:
   ```bash
sudo apt install wget
   ```

3. **Download Installer**:
   ```bash
wget http://github.com/utmstack/UTMStack/releases/latest/download/installer
```bash

4. **Switch to Root User**:
   ```bash
sudo su
   ```

5. **Set Execution Permissions**:
   ```bash
chmod +x installer
```bash

### Running the Installation

- Execute the installer:
  ```bash
./installer
  ```

After installation, use `admin` as the username and the password generated during installation to log in. The password and other configurations can be found in `/root/utmstack.yml`.

&gt; **Access Note**: Use HTTPS with your server name or IP to access the login page.

## Network Configuration

### Required Ports

- **22/TCP**: Secure Shell (SSH) - Recommended to allow only from admin workstation.
- **80/TCP**: UTMStack Web-based GUI Redirector - Recommended to allow only from admin and security analyst workstations.
- **443/TCP**: UTMStack Web-based GUI - Recommended to allow only from admin and security analyst workstations.
- **9090/TCP**: Cockpit Web-based Interface for Servers - Recommended to allow only from admin workstation.

Additional ports may be required for UTMStack's integrations. Follow security recommendations in the integration guide if available.
